# Technical Spec Pack v1

This folder is the canonical implementation pack for the JARVIS Project Charter v1.2.

## Scope
- Single-user, agentic Windows JARVIS
- Always-on timeline memory
- Groq-first intelligence pipeline
- React localhost control center + Electron orb shell
- MCP + Python tool ecosystem
- High-risk action approvals enforced

## Folder Layout
- `CHARTER.md`: finalized charter baseline
- `contracts/api-contracts.md`: service APIs and payload contracts
- `policies/risk-policy.md`: autonomy and approval policy details
- `schemas/tool.schema.json`
- `schemas/policy.schema.json`
- `schemas/memory_event.schema.json`
- `schemas/workflow.schema.json`
- `schemas/approval_request.schema.json`

## Build Order
1. Foundation runtime + UI shell
2. Voice stack
3. Tool layer
4. Headless integrations
5. Memory layer
6. Biometrics and policy enforcement
7. Proactivity + learn-from-me
8. Hardening and release channels

## Non-Goals (v1)
- Multi-user tenancy
- Obsidian integration
- Fully offline large-model orchestration
